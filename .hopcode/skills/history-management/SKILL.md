# History Management System

HopCode stores the full conversation history in a `HistoryVault` Î“Ă‡Ă¶ a plain
in-memory class that lives **outside React state**. Only a sliding window of
up to **2 000 items** is pushed into React state at any time. This prevents
unbounded React re-renders and heap OOM crashes during long sessions.

## HistoryItem types

`HistoryItem` is a discriminated union on `type`. Common values:

| `type`                 | Key fields                       | Searchable via               |
| ---------------------- | -------------------------------- | ---------------------------- |
| `user`                 | `text`                           | `text`                       |
| `gemini`               | `text`                           | `text`                       |
| `info`                 | `text`                           | `text`                       |
| `error`                | `text`                           | `text`                       |
| `warning`              | `text`                           | `text`                       |
| `tool_use`             | `name`, `input`                  | Î“Ă‡Ă¶ (not indexed)            |
| `tool_result`          | `content`                        | Î“Ă‡Ă¶                          |
| `away_recap`           | `text`                           | `text`                       |
| `memory_saved`         | `summary`                        | `summary`                    |
| `history_context_note` | `text`, `windowFrom`, `windowTo` | synthetic Î“Ă‡Ă¶ never in vault |

`history_context_note` items are **synthetic** Î“Ă‡Ă¶ generated by the vault and
prepended to the window slice when `windowStart > 0`. They are never stored
in `_items` and are never returned by `search()`.

Every `HistoryItem` has an `id: number`. Real items get positive IDs
(`Date.now() + counter`). Synthetic context notes get **negative** IDs to
prevent collisions.

## Reading history

### From a slash command (`CommandContext`)

```typescript
// context.ui is CommandContext['ui']

// The visible window (what the user sees)
const visibleItems = context.ui.history; // NOT directly available here Î“Ă‡Ă¶
// history is in React state, not in CommandContext.

// Window position
const info = context.ui.windowInfo;
// info.total        Î“Ă‡Ă¶ all items in the vault
// info.windowStart  Î“Ă‡Ă¶ index of first visible item
// info.windowEnd    Î“Ă‡Ă¶ index after last visible item
// info.hasOlder     Î“Ă‡Ă¶ can go further back
// info.hasNewer     Î“Ă‡Ă¶ can go forward to recent

// Navigate
if (context.ui.canLoadOlderHistory) context.ui.loadOlderHistory?.();
if (context.ui.canLoadNewerHistory) context.ui.loadNewerHistory?.();

// Jump to any global index (centers the window on that item)
context.ui.jumpToSearchResult?.(globalIndex);
```

### From a React component

```typescript
import { useUIState } from '../contexts/UIStateContext.js';

function MyComponent() {
  const { historyManager } = useUIState();

  // Current window
  const { history, windowInfo } = historyManager;

  // Navigate
  if (historyManager.canLoadOlderHistory) historyManager.loadOlderHistory();
  if (historyManager.canLoadNewerHistory) historyManager.loadNewerHistory();
}
```

### Directly from the vault (advanced)

The vault ref lives inside `useHistoryManager` and is not exported. If you
need raw vault access, read from `historyManager` on the UIState context. For
most cases the hook API is sufficient.

## Searching history

Search scans **every item in the vault** (not just the visible window), up to
200 results, case-insensitive substring match.

### Via slash command

```
/search <query>
/history search <query>
```

Output lists each match with:

- Global index (use `/history jump <n>` to navigate there)
- Item type
- A contextual excerpt with the match highlighted in `Î“Ă‡ÂŞsurrounding textÎ“Ă‡ÂŞ`

### Programmatically

```typescript
// In a slash command
const results = context.ui.searchHistory?.('OOM fix') ?? [];

results.forEach(({ item, globalIndex, matchExcerpt }) => {
  console.log(`#${globalIndex} [${item.type}] ${matchExcerpt}`);
});

// Jump the window to the first result
if (results.length > 0) {
  context.ui.jumpToSearchResult?.(results[0].globalIndex);
}
```

```typescript
// HistorySearchResult shape
interface HistorySearchResult {
  item: HistoryItem; // the matched item
  globalIndex: number; // position in _items[] (0-based)
  matchExcerpt: string; // "Î“Ă‡ÂŞbefore MATCH afterÎ“Ă‡ÂŞ"
}
```

`getSearchableText(item)` is used internally Î“Ă‡Ă¶ it reads `item.text` first,
then `item.summary`, returning `''` for types that have neither. Only items
with a non-empty searchable string are candidates.

## Window navigation

```
/history older    Î“Ă‡Ă¶ shift window WINDOW_SIZE items toward the past
/history newer    Î“Ă‡Ă¶ shift window WINDOW_SIZE items toward the present
/history info     Î“Ă‡Ă¶ show current position (windowStart, windowEnd, total)
/history jump <n> Î“Ă‡Ă¶ center window on item at global index n
```

`WINDOW_SIZE = 2_000` (constant in `HistoryVault.ts`).

Window always auto-advances to the live edge as new items arrive, so during
normal use the user never needs to navigate. Navigation is only needed when
reviewing past history.

## Context notes

When `windowStart > 0` (i.e. the user has scrolled back to see older items),
the vault prepends a synthetic `history_context_note` item to the window
slice. This note:

- Scans the **hidden older items** (newest-first, up to 6 excerpts)
- Includes only items of types: `user`, `gemini`, `info`, `error`, `warning`,
  `away_recap`
- Formats each excerpt as `[type] first 180 charsÎ“Ă‡ÂŞ`
- Is cached by `_windowStart` Î“Ă‡Ă¶ rebuilt only when older items are mutated

**The AI sees context notes in the rendered history**, so it has awareness of
what happened before the visible window even when those items are not in React
state.

Rendered appearance in the TUI: a single-border box with dim text, reading
`Î“ĂĄĂ¦ Older history (items 0Î“Ă‡Ă´N not shown)` followed by the excerpts.

## Adding items (for feature development)

```typescript
// addItem returns the new item's id
const id = historyManager.addItem(
  {
    type: 'info',
    text: 'Something happened',
  },
  Date.now(),
);

// Update an existing item (use sparingly Î“Ă‡Ă¶ forces re-render)
historyManager.updateItem(id, { text: 'Updated text' });

// Load a saved history (e.g. /resume)
historyManager.loadHistory(savedItems);

// Truncate from a given id onward (inclusive)
historyManager.truncateToItem(id);
```

Duplicate consecutive `user` items with identical `text` are silently dropped
by `push()` Î“Ă‡Ă¶ this is intentional to prevent double-sends.

## Common pitfalls

| Pitfall                                                             | Correct approach                                                                                                                                                                             |
| ------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Reading `history[history.length - 1]` to detect the last real item  | Use `context.ui.windowInfo` or call `vault.getLastItem()` Î“Ă‡Ă¶ the last item in `history[]` may be a synthetic context note                                                                   |
| Expecting `history.length` to equal total items                     | `history.length Î“Ă«Ă± WINDOW_SIZE + 1`. Use `windowInfo.total` for the real count                                                                                                              |
| Calling `searchHistory` and expecting only visible items            | `searchHistory` scans ALL vault items Î“Ă‡Ă¶ results may have `globalIndex` outside the current window                                                                                          |
| Comparing `globalIndex` to `history` array index                    | `globalIndex` is a vault-level index; the `history` array starts at `windowStart`, so `history[i]` corresponds to vault index `windowStart + i` (approximately, ignoring the prepended note) |
| Adding a new history item type without updating `getSearchableText` | Search will silently skip items of that type. Add a case to `getSearchableText` in `HistoryVault.ts`                                                                                         |
